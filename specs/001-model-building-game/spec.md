# Feature Specification: 模型建構遊戲 (Model Building Game)

**Feature Branch**: `001-model-building-game`
**Created**: 2026-03-13
**Status**: Draft
**Input**: User description: "模型建構遊戲"

## Constitution Alignment *(mandatory)*

- **CA-001 最小設計**: 本功能專注於核心建構互動（選件、放置、組合），不實作進階 3D 渲染或多人聯線功能，維持最小可交付範圍。
- **CA-002 文件節制**: 僅維護本 spec.md 及必要的 checklist；不另行建立變更總結文件。
- **CA-003 TDD 可測性**: 每個功能需求均定義「先失敗、後通過」的驗收情境，確保可測試驅動開發。
- **CA-004 Git 可追溯性**: 在規格完成、規劃完成、實作完成三個節點各建立 git checkpoint，確保進度可追溯。
- **CA-005 Implement 保護**: 實作階段不得刪除或修改本 spec.md；tasks 勾選需反映實際完成狀態。
- **CA-006 網站預設交付**: 本遊戲以可部署至 GitHub Pages 的靜態前端網站為交付目標，無需後端伺服器。

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 基礎模型建構 (Priority: P1)

玩家進入遊戲後，可從零件庫中選取零件並將其放置於建構區域，逐步組裝成完整模型。每個零件都可以在指定的合法位置放置，並在組裝完成後獲得視覺回饋（例如完成動畫或成就提示）。

**Why this priority**: 這是遊戲的核心互動，沒有零件選取與放置功能，遊戲即無法運作。此故事交付後即可構成完整的最小可玩版本（MVP）。

**Independent Test**: 開啟遊戲頁面，從零件庫選取至少一個零件並放置於建構區，確認零件正確顯示於所放位置，並在完成全部必要零件後觀察到完成回饋。

**Acceptance Scenarios**:

1. **Given** 玩家位於遊戲主畫面，**When** 點選零件庫中的任一零件，**Then** 該零件以「已選取」狀態高亮顯示，且游標附近出現預覽圖。
2. **Given** 玩家已選取一個零件，**When** 點選建構區域中的合法放置格，**Then** 零件被放置於該格，零件庫中該零件數量減一（若有數量限制），且放置後立即更新畫面。
3. **Given** 玩家已完成模型所有必要零件的放置，**When** 系統偵測到模型完整，**Then** 出現完成動畫或成就提示，並顯示玩家得分。
4. **Given** 玩家嘗試將零件放置於非法格（已有零件或不相容位置），**When** 點選該格，**Then** 放置操作被拒絕並顯示友善提示訊息，零件回到待放置狀態。

---

### User Story 2 - 模型選擇與進度管理 (Priority: P2)

玩家可在開始遊戲前瀏覽可用的模型目錄，選擇想要建構的模型（例如：房屋、車輛、機器人）。遊戲自動儲存進度，玩家可在稍後繼續未完成的模型。

**Why this priority**: 讓玩家能選擇目標並保存進度，大幅提升重複遊玩意願；但在無此功能時仍可玩單一預設模型。

**Independent Test**: 開啟遊戲，確認出現模型選擇畫面（至少 3 種模型可選），選取一個模型後開始建構，離開頁面後重新進入，確認先前的建構進度被保留。

**Acceptance Scenarios**:

1. **Given** 玩家首次進入遊戲，**When** 遊戲載入完成，**Then** 顯示至少 3 種不同的可建構模型供選擇，每種模型附有縮圖與名稱。
2. **Given** 玩家選擇了某個模型並開始建構，**When** 玩家關閉或離開遊戲頁面後重新進入，**Then** 自動載入上次的建構進度，玩家可繼續未完成的組裝。
3. **Given** 玩家已完成一個模型，**When** 返回主選單，**Then** 已完成的模型以「已完成」標示顯示，玩家可選擇建構下一個模型。

---

### User Story 3 - 得分與成就系統 (Priority: P3)

玩家完成模型後獲得得分，系統記錄玩家的最高分與完成時間，並提供成就徽章鼓勵探索不同模型或在時限內完成建構。

**Why this priority**: 成就系統增加遊戲深度與重玩動機，但並非核心遊玩功能，可在基礎功能完善後擴充。

**Independent Test**: 完成一個模型，確認顯示分數，並在「成就」頁面看到對應的徽章已解鎖。

**Acceptance Scenarios**:

1. **Given** 玩家完成模型建構，**When** 系統顯示完成畫面，**Then** 同時顯示本次得分（依完成速度與準確度計算）及歷史最高分。
2. **Given** 玩家首次完成特定模型，**When** 完成畫面出現，**Then** 解鎖對應成就徽章並伴隨動畫提示。
3. **Given** 玩家查看成就頁面，**When** 選單中點選「成就」，**Then** 顯示所有成就徽章列表，已解鎖者亮色顯示、未解鎖者灰色顯示。

---

### Edge Cases

- 當玩家嘗試同時放置超過允許數量的相同零件時，系統拒絕放置並提示當前可用數量。
- 當本機儲存空間不足或被清除時，系統顯示提示並允許玩家重新開始，不造成應用程式崩潰。
- 當玩家在建構途中重新整理頁面，自動儲存的進度應能正確還原至最後一次操作。
- 當玩家在零件選取後按下取消或點擊其他區域，已選取的零件應回到未選取狀態。
- 當模型目錄全部完成後，系統顯示全部完成的慶賀畫面，並提供重玩選項。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 系統必須提供零件庫面板，顯示當前所選模型所需的全部零件及其可用數量。
- **FR-002**: 系統必須提供可互動的建構區域（畫布），讓玩家以點選方式放置零件。
- **FR-003**: 系統必須驗證每次零件放置操作的合法性，並在非法放置時提供即時回饋。
- **FR-004**: 系統必須在玩家完成完整模型時自動偵測並觸發完成流程（動畫、得分計算）。
- **FR-005**: 系統必須自動儲存玩家的建構進度至本機，確保頁面重新載入後可還原。
- **FR-006**: 系統必須提供至少 3 種不同主題的可建構模型（例如：房屋、車輛、機器人）。
- **FR-007**: 系統必須記錄並顯示每個模型的最高分與完成時間。
- **FR-008**: 系統必須提供成就徽章系統，在特定里程碑（首次完成某模型、限時完成等）時解鎖。
- **FR-009**: 系統必須可在不需後端伺服器的情況下於主流瀏覽器中完整運作。
- **FR-010**: 玩家必須能夠撤銷（Undo）最近一次零件放置操作。

### Key Entities

- **模型 (Model)**: 代表一個可建構的目標物件，包含名稱、縮圖、完整零件清單及合法放置規則。
- **零件 (Part)**: 模型的組成單元，具有類型、形狀、可放置位置限制及圖示。
- **建構區域 (Build Canvas)**: 玩家放置零件的互動介面，以格狀或自由形式定義合法位置。
- **建構進度 (Build Progress)**: 記錄玩家在特定模型上已放置的零件狀態，支援儲存與還原。
- **得分記錄 (Score Record)**: 記錄玩家完成各模型的得分、時間與成就解鎖狀態。
- **成就 (Achievement)**: 代表特定遊玩里程碑，包含名稱、解鎖條件與圖示。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 玩家從選取模型到放置第一個零件的時間不超過 30 秒（操作流暢度）。
- **SC-002**: 完成一個完整模型（含至少 10 個零件）的整體建構流程可在 10 分鐘內完成。
- **SC-003**: 90% 的零件放置操作在玩家點選後 200 毫秒內完成視覺更新（回應速度）。
- **SC-004**: 建構進度在頁面重新載入後 5 秒內完整還原，還原準確率達 100%（進度儲存可靠性）。
- **SC-005**: 遊戲在首次開啟時的載入時間不超過 3 秒（在一般網路環境下）。
- **SC-006**: 玩家首次遊玩無需任何說明即可完成第一個模型的建構（零學習曲線直覺性）。
- **SC-007**: 遊戲提供至少 3 種可建構模型，每種模型包含至少 8 個零件，以保障足夠的遊玩內容量。

## Assumptions

- 本遊戲以瀏覽器為主要平台，採用靜態網站形式部署至 GitHub Pages，不需後端服務。
- 玩家進度使用瀏覽器本機儲存（localStorage 或等效機制）保存，不需帳號登入。
- 模型以 2D 視角呈現，採用格狀建構系統（非自由拖曳式），以降低實作複雜度。
- 零件圖示採用簡單幾何圖形或像素風格，無需外部素材授權。
- 目標使用者年齡層為 8 歲以上的一般玩家，介面語言以中文為主。
- 遊戲支援現代主流瀏覽器（最近兩個主要版本），不需支援 IE。
- 單次遊玩（完成一個模型）不需登入，成就與最高分記錄於本機。
